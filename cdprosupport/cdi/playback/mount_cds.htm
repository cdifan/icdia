Changing CDs in CD-i Playback for Windows

When trying to change discs from the MediaMogul startup disc to a different disc or disc image containing, for example, mpeg assets, some PC's have a problem recognizing the second disc. This problem has a workaround: The key factor is that you have to do a dir of the cd between ejecting the MM cd and opening the image file. This clears things out somehow. This problem is fixed in the new version, now called MediaAuthor 1.5.1.

The workaround steps for CD-i Playback 2.2.x are as follows:

  1. Start MediaMogul.
  2. At the MM shell, choose CD-RTOS.
  3. eject the MM cd using the eject menu command or button in Playback.
  4. type:
    dir /cd
    in CD-RTOS. (No quotes. Actually getting a directory of...nothing.) Repeat this command until you get a #000:244 error. This error indicates that the MM cd has been cleared out of memory completely.
  5. Load the cd image with the Open command or ,if applicable,
    replace the MediaMogul disc with the second disc and close the drawer.
  6. Mount the second disc by clicking the mount button once (on some systems, twice).
  7. type:
    dir /cd/GREEN
    (e.g.) to confirm that it's reading the disc/disc image properly. When it's not working, it sees subdirectories as files, and won't open them or give a directory of them.
For MediaAuthor 1.5.1, the steps for changing cd's are:
  1. Start MediaMogul
  2. Click on CD-RTOS to get an OS9 shell.
  3. At the MM shell CD-i> prompt, type mm_eject. The MediaMogul cd should eject.
  4. Replace the cd with the second disc or open an image file. You should now have access to it.
Neither of these procedures are strictly necessary, as it is now possible to boot off the hard drive without the MediaMogul cd present. See the No CD page.
