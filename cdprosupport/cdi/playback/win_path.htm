Pathing issues with CDi Playback for Windows

  • CD-i Playback must be installed on C:\ drive, or it won't work.

  • The MediaMogul directory can be on any drive.

  • When specifying OS9 virtual drives within CDi Playback, you cannot specify an entire PC drive with an OS9 device descriptor. You can only associate the driver name with a folder. For example, you can specify "d:\dir_name\" as your "h1" drive, but not d:\ itself.

  • MediaMogul will not install correctly if the path to /h0 is incorrectly set in C:\Windows\wcdiplay.ini.

    All OS9 path aliases in wcdiplay.ini must end with a trailing slash , e.g.

    [RBFM]
    H0=C:\MM\
    H1=D:\FILES\
    DD=A:\

    If a trailing slash is left out of the H0 line, all MediaMogul files will be installed at the root, and have a name starting with "MM"!
