Errors starting CDIPlayback for Windows



The changes you need to make in the AUTOEXEC.BAT and CONFIG.SYS files when installing CDIPlayback for Windows are outlined in the Installation section of the manual. Additionally, under some circumstances you may need to make changes to the WCDIPLAY.INI file located in your WINDOWS directory.

If you get errors when starting CDIPlayback, you may need to edit the WCDIPLAY.INI file. The errors can range from a message that CDIPlayback couldn't be started and needs to be configured to a General Protection Fault, and can vary from installation to installation.

The WCDIPLAY.INI file can be opened with any text editor, such as Notepad. In the file make sure the base address is set to the one your board is actually set to. This is the lower (or first) of the memory addresses in the "X=nnnn-nnnn" statement that you entered in the EMM386.EXE DEVICE specification in your CONFIG.SYS file.

NOTE: The base address for the board is located in two places in the WCDIPLAY.INI file, and should be changed in both locations.

Additionally, you should make sure that the COM port setting in the WCDIPLAY.INI file is set to the COM port you are actually using.

After making these changes, save the WCDIPLAY.INI file, exit your text processor, and restart Windows. You should now be able to start CDIPlayback.
