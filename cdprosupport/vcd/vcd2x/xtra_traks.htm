Redundant Tracks in PC VCD 2.0 Toolkit


 

     If the "Autoinclude MPEGAV files as tracks" checkbox is checked in the Preferences dialog of PC VCD 2.0 Toolkit, MPEGAV assets are added as separate tracks in the Track Manager each time they are added as assets in the Asset Manager. This also applies if the same MPEGAV asset is accidentally added two(or more) times to the Asset Manager. The asset will only be displayed once in the Asset Manager, but will be listed as a separate track in the Tracks Manager as many times as it is added. This can significantly reduce disc space available. To avoid this problem either:
 

  1. Be careful not to add the same asset more than once to the Asset Manager.
  2. Check the Tracks Manager for duplicate MPEGAV files before building.
  3. Manually remove all duplicate tracks in the Tracks Manager.
  4. Turn off the "Autoinclude" preference. If you do this, however, be sure to add the MPEGAV files manually to the Tracks Manager if you desire VCD1.X compatibility.
