VCD and CD-i CDR Recording Under Windows 95/NT

Both WinOnCD and Gear for Windows 95 work with our disc images, but you do have to follow a certain procedure. First of all, find in your build folder the file that has a ".cd" extension in its name. This file is the disc image file. All the other files in the build directory are either temporary files used during the build, or are associated with the disc image file. For the purpose of burning a disc, though, the disc image file is the only file to be concerned with. The disc image file contains everything you included in your project: the assets (stills, motion, audio), the menuing, & other files. In fact, if you have a CD-i Authoring board, or CD-i Emulator, you can emulate that file just as though you were playing a real VideoCD/CD-i disc.

In order to use Gear or WinOnCD to burn a VCD or CD-i disc, there are two kinds of disc images to consider:

Single-Track Disc Images
Synopsis(Step-by-step procedure)
To use the disc image file as is with Gear or WinOnCD, you must build a single-track disc image. It should be remembered that:

  • Standard CD-i disc images are always single-track.

  • VCD 1.x discs are always multi-track.

  • VCD 2.0 disc images usually have multiple tracks,
    (though playable single-track discs can be made.)
VCD 2.0 Toolkit Considerations:
If you are using chaptermarks or desire VCD 1.x compatibility you must place the MPEGAV files on the disc as separate tracks--see the Multitrack disc instructions. To make a single-track VCD 2.0 disc image using the PC Toolkit, you must uncheck the "Auto include VCD MPEG files as Tracks" option in the Preferences of VCD Toolkit. Then, all MPEG clips you add to your project will only be added to the first (and only) track. The Mac version of the Toolkit will not add MPEGAV as tracks automatically, it must be done manually, so there are no added steps for creating single-track images. If you already have made a project with the "Auto include" option checked, or have manually added tracks, you may need to manually remove the extra MPEGAV tracks in the Tracks Manager. When you build the project, you will get a warning message about the White Book requiring at least one MPEGAV track: you can safely ignore this message. Although the disc produced will not be precisely to the VCD specification, it should play fine on all CD-i players, and VCD 2.0 players.

 

Multi-Track Disc Images
Synopsis(Step-by-step procedure)
There are four ways currently to burn a multi-track VCD disc with our disc images:

  1. use CD Record. This is Windows 3.x-only software. It relies on the ".toc" file, also in the build folder. The .toc file (Table Of Contents) describes the structure of the disc, including the tracks.
  2. use Toast (or our OEM'd version, CD-IT!ALL). This is Mac-only software, which interprets our disc images directly.
  3. use Disctool(or the command-line version, CDR521) and a CD-i E2 Emulator with recording capability.
  4. use our Splitter utility. This utility splits our multi-track disc images into multiple files, so that Gear or WinOnCD can record them. Then follow the Step-by-step procedure.

Elektroson, the makers of Gear, are currently working on a version of Gear for Windows 95 which will utilize our .toc files in order to burn multi-track images. This version may be available in the near future.

The procedures for recording disc images with Gear or WinOnCD follow.

Recording Procedures

RECORDING SINGLE-TRACK VCD or CD-i DISC IMAGES WITH GEAR

(goto Multi-Track Images)
  1. Select "External Disc Image" from the CD Type & Settings pulldown menu
  2. Click the Open button, browse to the build folder, & select the ".cd" file. The External Image Parameters dialog should appear.
  3. Select "CD-ROM XA sector size 2352 scrambled, with 2 sec pregap" from the Image Type pulldown menu in the dialog, and click OK.
  4. Burn the disc.

RECORDING SINGLE-TRACK VCD or CD-i DISC IMAGES WITH WinOnCD

(goto Multi-Track Images)
  1. Select the "Raw Data" type from the New Project dialog
  2. Double-click on the "Raw Data" CD-ROM track in the track window: The Raw File Properties dialog should appear.
  3. Click the Select button, browse to the build folder, & select the ".cd" file.
  4. Select "CD-ROM XA" for Track format.
  5. Select "2352 bytes data scrambled" for File format.
  6. Enter "150" in theStart writing at block offset field and click Ok.
  7. Burn the disc.

RECORDING MULTI-TRACK VCD DISC IMAGES WITH GEAR

(goto Single-Track Images)
  1. Use the splitter.exe utility to separate a multi-track disc image into multiple individual track files.
  2. Create a cue sheet file listing all the tracks.
  3. Select "Cue Sheet" from the CD Type & Settings pulldown menu
  4. Click the Open button, browse to the cue sheet file, & select it. You will not be able to edit this cue sheet within Gear, but it should burn a disc properly.
  5. Burn the disc.

RECORDING MULTI-TRACK VCD DISC IMAGES WITH WinOnCD

(goto Single-Track Images)
  1. Use the splitter.exe utility to separate a multi-track disc image into multiple individual track files.
  2. Select the "Raw Data" type from the New Project dialog
  3. Double-click on the "Raw Data" CD-ROM track in the track window: The Raw File Properties dialog should appear.
  4. Click the Select button, browse to the build folder, & select the first track file.
  5. Select "CD-ROM XA" for Track format.
  6. Select "2352 bytes data scrambled" for File format.
  7. Enter "150" in theStart writing at block offset field and click Ok.
  8. Highlight the last track, then select "Insert" from the Edit menu, or hit the Insert key.
  9. Choose "Raw Data" again in the Insert Track dialog, be sure that after Selection is selected, then click Ok.
  10. Repeat steps 3 through 7 on the newly inserted track, using the next Track file.
  11. Repeat steps 8,9, and 10 using the subsequent Track files until all Track files created by the splitter application have been used.
  12. Burn the disc.
